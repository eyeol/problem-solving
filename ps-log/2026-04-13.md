## Floyd's Cycle Detection

토끼와 거북이 알고리즘이라고도 불리는 순환 탐지 알고리즘이다.

LC 150 문제 중에 141. Linked List Cycle 를 풀다가 알게 된 알고리즘

```
Given head, the head of a linked list, determine if the linked list has a cycle in it.

There is a cycle in a linked list if there is some node in the list that can be reached again by continuously following the next pointer. Internally, pos is used to denote the index of the node that tail's next pointer is connected to. Note that pos is not passed as a parameter.

Return true if there is a cycle in the linked list. Otherwise, return false.
```

```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, x):
#         self.val = x
#         self.next = None

class Solution:
    def hasCycle(self, head: Optional[ListNode]) -> bool:

        visited = set()
        curr = head

        while curr:
            if curr in visited:
                return True
            # 방문한적 없으면
            visited.add(curr)
            curr = curr.next

        return False
```

처음에는 visited를 set으로 써야한다는것도 몰라서 어버버 거리다가<br>
set을 쓰면 객체의 메모리 주소 기반으로 방문 여부를 체크할 수 있다는걸 배움

이걸 몰라서 딕셔너리나 리스트로 어떻게 해야하는거지?? 하고 있었음..

근데 이렇게 풀고나니

```
Follow up: Can you solve it using O(1) (i.e. constant) memory?
```

..? 상수 메모리로 풀어볼 수 있겠냐는 도발을 봄..<br>

찾아보니 토끼와 거북이 알고리즘이라는게 있었다<br>

빠른 포인터(토끼)와 느린 포인터(거북이)를 헤드에서 출발시켜서<br>
둘이 만나면 사이클이 있는거, 안만나면 사이클이 없는거

둘이 만났다면 만난 시점에서 헤드에서 포인터 하나를 출발시켜서<br>
사이클 시작점도 특정할 수 있고, 사이클 시작점을 안 후에 사이클 길이도 측정 가능함

나중에 블로그에 짤막하게라도 정리하자

## 완전탐색이랑 코드트리

친구가 알려준 코드트리에서 갭체크를 해보니까<br>
완전탐색이 부족하다고.. 허접 판정을 받았다..

코드트리 결제까지 할 정도는 아닌거 같고 다른 주제는 괜찮은데<br>
완전 탐색 부분은 따로 공부해서 체계적으로 정리하면 좋을것 같다.

### 완전탐색 유형

- 브루트포스: for문으로 전부 돌림
- 순열: N개를 순서 있게 나열 (N!)
- 조합: N개 중 R개 순서 없이 선택
- 부분집합: 각 원소 포함/미포함 (2^N)
- 백트래킹: 위 탐색 + 가지치기 (조건 안 맞으면 되돌아감)

이것도 블로그에 정리하면서 유형별 문제들 풀어보자
